# UI_UX

## Question

Quelle expérience utilisateur voulons-nous offrir ?

---

# Principe

L'utilisateur ne doit jamais avoir l'impression d'utiliser un catalogue de pièces.

Il doit avoir l'impression de discuter avec un expert automobile.

La plateforme doit être :

- simple ;
- rapide ;
- rassurante ;
- explicite.

---

# Page d'accueil

La première page présente deux choix.

## Je connais la pièce

Recherche directe.

Le client peut saisir :

- une référence ;
- le nom d'une pièce ;
- une plaque ;
- un VIN.

---

## J'ai un problème

Le client démarre une conversation avec l'assistant IA.

Exemples :

- Ma voiture ne démarre plus.
- J'entends un bruit.
- Un voyant est allumé.
- Je dois faire l'entretien.

---

# Conversation IA

L'assistant :

- comprend la demande ;
- identifie le véhicule ;
- pose uniquement les questions nécessaires ;
- explique son raisonnement.

---

# Résultats

Chaque résultat affiche :

- la pièce proposée ;
- la raison de la recommandation ;
- le niveau de confiance ;
- le prix ;
- le stock ;
- le délai ;
- le magasin.

---

# Réservation

Le client peut réserver en quelques clics.

Avant la validation, un récapitulatif est affiché.

---

# Principes UX

Toujours expliquer.

Jamais surprendre.

Toujours permettre de revenir en arrière.

Ne jamais afficher d'informations inutiles.

Réduire le nombre de clics.

La confiance est plus importante que la vitesse.

---

# Objectif

Un utilisateur qui découvre TapiecesAuto doit comprendre le fonctionnement de la plateforme en moins d'une minute.
