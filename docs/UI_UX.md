# MVP

## Question

Quelle est la plus petite version de TapiecesAuto capable d'apporter une vraie valeur à un client ?

---

# Objectif

Mettre en ligne une première version simple, fiable et utilisable le plus rapidement possible.

Le MVP doit permettre de valider le concept auprès de vrais utilisateurs avant d'ajouter des fonctionnalités avancées.

---

# Fonctionnalités incluses

## Identification du véhicule

Le client peut :

- saisir son numéro de plaque ;
- saisir son numéro VIN ;
- sélectionner manuellement son véhicule.

---

## Recherche

Deux parcours sont disponibles :

### Je connais la pièce

Recherche directe.

### J'ai un problème

Conversation avec l'assistant IA.

---

## Assistant IA

L'IA doit pouvoir :

- comprendre la demande ;
- identifier le véhicule ;
- poser les bonnes questions ;
- expliquer son raisonnement ;
- recommander une ou plusieurs pièces compatibles ;
- indiquer son niveau de confiance.

---

## Résultats

Chaque résultat affiche :

- nom de la pièce ;
- fabricant ;
- prix ;
- disponibilité ;
- niveau de compatibilité ;
- justification de la recommandation.

---

## Réservation

Le client peut :

- choisir un magasin ;
- réserver la pièce ;
- recevoir une confirmation.

---

# Fonctionnalités exclues du MVP

Ces éléments seront développés plus tard :

- application mobile ;
- reconnaissance vidéo ;
- reconnaissance vocale ;
- historique complet du véhicule ;
- connexion OBD ;
- maintenance prédictive ;
- recommandations personnalisées ;
- marketplace complète.

---

# Critères de réussite

Le MVP est considéré comme réussi si un utilisateur peut :

1. identifier son véhicule ;
2. trouver une pièce compatible ;
3. comprendre pourquoi cette pièce est proposée ;
4. réserver cette pièce en quelques minutes.

---

# Philosophie

Le MVP ne cherche pas à proposer le plus grand nombre de fonctionnalités.

Il cherche à offrir la meilleure expérience possible pour résoudre un problème automobile rapidement et avec confiance.
